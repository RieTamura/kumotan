プライバシーポリシー

くもたん（以下「本アプリ」といいます。）の運営者（以下「運営者」といいます。）は、本アプリにおけるユーザーの個人情報の取扱いについて、以下のとおりプライバシーポリシー（以下「本ポリシー」といいます。）を定めます。

第1条（個人情報の定義）

「個人情報」とは、個人情報保護法にいう「個人情報」を指すものとし、生存する個人に関する情報であって、当該情報に含まれる氏名、生年月日、住所、電話番号、連絡先その他の記述等により特定の個人を識別できる情報（個人識別符号となるものを含む）を指します。

第2条（収集する情報）

本アプリは、以下の情報を取得する場合があります。

1. Blueskyアカウント情報

Bluesky OAuth認証を通じて以下の情報を取得します。

  • ユーザーのDID（分散型識別子）
  • ハンドル名（ユーザー名）
  • 表示名、アバター画像、プロフィール情報

2. 端末内に保存されるデータ

以下のデータはユーザーの端末内にのみ保存され、運営者のサーバーに送信されることはありません。

  • 登録した単語データ（英単語、日本語訳、定義、登録元の投稿情報）
  • 学習統計データ（日別の学習数、クイズ履歴等）
  • アプリ設定（テーマ、言語、チュートリアル完了状態等）
  • 外部サービスのAPIキー（DeepL API Key、Yahoo! JAPAN Client ID）

3. PDSに同期されるデータ

AT Protocolの仕組みに基づき、以下のデータがユーザー自身のPDS（Personal Data Server）に同期されます。

  • 登録した単語データ（英単語、日本語訳、定義、既読状態、登録元の投稿URL等）
  • 学習セッション情報

4. 外部サービスに送信されるデータ

本アプリの機能を提供するため、以下のデータが外部サービスに送信されます。

  • Bluesky / AT Protocol: タイムライン取得リクエスト、投稿内容（進捗シェア時）、画像データ（投稿添付時）
  • DeepL API: 翻訳対象のテキスト（ユーザーがAPIキーを設定した場合のみ）
  • Yahoo! JAPAN テキスト解析API: 解析対象の日本語テキスト（ユーザーがClient IDを設定した場合のみ）
  • Free Dictionary API: 検索する英単語
  • Railway（運営者管理サーバー）: Bluesky DID、Expo Push Token、通知設定（プッシュ通知を有効にした場合のみ）
  • Expo Push Notification Service: Expo Push Token、通知タイトル・本文（プッシュ通知送信時）
  • Bluesky Jetstream（公式）: 送信なし（イベント受信のみ）

5. プッシュ通知トークン

ユーザーがプッシュ通知を有効にした場合、以下の情報が運営者管理サーバー（Railway）に送信・保存されます。

  • Expo Push Token（デバイス識別子。Expo Push Notificationサービスが発行する通知配信用トークン）
  • Bluesky DID（通知の宛先を特定するための公開識別子）
  • 通知種別の設定（いいね・返信・メンション・リポスト・フォローの各ON/OFF）

これらの情報はプッシュ通知の送信目的にのみ使用し、それ以外の目的には使用しません。プッシュ通知を無効にした場合、または本アプリをアンインストールした場合は、サーバーから削除されます。

6. フィードバック情報

ユーザーがアプリ内のフィードバック機能を利用した場合、以下の情報が送信されます。

  • フィードバックの種類（バグ報告、機能要望、辞書修正等）
  • フィードバックの内容（件名、説明、コメント）
  • 関連する単語情報や投稿URL（該当する場合）

第3条（情報の利用目的）

運営者が情報を利用する目的は、以下のとおりです。

  1. 本アプリの提供・運営のため
  2. ユーザー認証およびBlueskyアカウントとの連携のため
  3. 単語データの保存・復元機能を提供するため
  4. 辞書検索、翻訳、テキスト解析等の学習支援機能を提供するため
  5. ユーザーからのフィードバックに対応し、本アプリを改善するため
  6. 本アプリの不具合の修正および機能改善のため
  7. プッシュ通知（Blueskyのソーシャルアクション通知・学習リマインダー）の送信のため

第4条（個人情報の第三者提供）

  1. 運営者は、次に掲げる場合を除いて、あらかじめユーザーの同意を得ることなく、第三者に個人情報を提供することはありません。
    • 人の生命、身体または財産の保護のために必要がある場合であって、本人の同意を得ることが困難であるとき
    • 公衆衛生の向上または児童の健全な育成の推進のために特に必要がある場合であって、本人の同意を得ることが困難であるとき
    • 国の機関もしくは地方公共団体またはその委託を受けた者が法令の定める事務を遂行することに対して協力する必要がある場合であって、本人の同意を得ることにより当該事務の遂行に支障を及ぼすおそれがあるとき
    • その他法令で認められる場合
  2. 本アプリは、ユーザーが設定した外部サービス（DeepL、Yahoo! JAPAN等）にデータを送信しますが、これはユーザーが自ら設定したAPIキーに基づく送信であり、運営者による第三者提供には該当しません。

第5条（安全管理措置）

  1. 運営者は、個人情報の漏えい、滅失またはき損の防止その他の個人情報の安全管理のために、以下の措置を講じています。
    • 認証トークンおよびAPIキーは、端末の暗号化ストレージ（Expo SecureStore）に保存します
    • Bluesky認証にはOAuth（PKCE + DPoP）を採用し、ユーザーのパスワードを取得・保存しません
    • 外部サービスとの通信はHTTPSによる暗号化通信を使用します

第6条（アナリティクス・トラッキング）

本アプリは、アクセス解析ツール、広告配信SDK、トラッキングツール等を使用しておりません。ユーザーの行動データを収集・分析する機能は搭載していません。

第7条（端末のアクセス権限）

本アプリは、以下の端末機能へのアクセス許可を求める場合があります。

権限 用途
写真ライブラリ 学習進捗の画像を保存・シェアするため、投稿に画像を添付するため
カメラ 投稿に添付する写真を撮影するため
通知 Blueskyのソーシャルアクション通知および学習リマインダーを受信するため

これらの権限は、ユーザーが該当する機能を利用する際にのみ求められ、許可しなくても本アプリの基本機能は利用できます。

第8条（データの削除）

  1. ユーザーは、本アプリの設定画面から「すべてのデータを削除」を実行することにより、端末内に保存されたすべての学習データを削除できます。
  2. ログアウトを行うことにより、端末内に保存された認証情報は削除されます。
  3. PDSに同期されたデータの削除については、ユーザー自身のPDS管理に委ねられます。
  4. 本アプリをアンインストールすることにより、端末内のすべてのデータが削除されます。
  5. 削除されたデータは復元できません。ユーザーは、削除前に必要に応じてデータエクスポート機能を利用し、バックアップを行ってください。

第9条（未成年者の利用）

本アプリは、特に未成年者を対象としたサービスではありませんが、未成年者が利用する場合は、保護者の同意のもとで利用することを推奨します。

第10条（プライバシーポリシーの変更）

  1. 運営者は、必要に応じて本ポリシーを変更することがあります。
  2. 本ポリシーの変更は、本アプリ内または運営者のウェブサイト上で通知するものとします。
  3. 変更後に本アプリの利用を継続した場合、ユーザーは変更後のポリシーに同意したものとみなします。

Privacy Policy

The operator (hereinafter the "Operator") of Kumotan (hereinafter "this App") establishes this Privacy Policy (hereinafter "this Policy") regarding the handling of users' personal information in this App.

Article 1 (Definition of Personal Information)

"Personal information" refers to "personal information" as defined in the Act on the Protection of Personal Information, meaning information about a living individual that can identify a specific individual by name, date of birth, address, telephone number, contact information, or other descriptions contained in the information (including personal identification codes).

Article 2 (Information We Collect)

This App may collect the following information:

1. Bluesky Account Information

The following information is obtained through Bluesky OAuth authentication:

  • User's DID (Decentralized Identifier)
  • Handle (username)
  • Display name, avatar image, profile information

2. Data Stored on Device

The following data is stored only on the User's device and is not sent to the Operator's servers:

  • Registered word data (English words, Japanese translations, definitions, source post information)
  • Learning statistics data (daily learning counts, quiz history, etc.)
  • App settings (theme, language, tutorial completion status, etc.)
  • External service API keys (DeepL API Key, Yahoo! JAPAN Client ID)

3. Data Synced to PDS

Based on the AT Protocol framework, the following data is synced to the User's own PDS (Personal Data Server):

  • Registered word data (English words, Japanese translations, definitions, read status, source post URLs, etc.)
  • Learning session information

4. Data Sent to External Services

To provide this App's features, the following data is sent to external services:

  • Bluesky / AT Protocol: Timeline retrieval requests, post content (when sharing progress), image data (when attaching to posts)
  • DeepL API: Text for translation (only when User has set API key)
  • Yahoo! JAPAN Text Analysis API: Japanese text for analysis (only when User has set Client ID)
  • Free Dictionary API: English words for lookup
  • Railway (Operator-managed server): Bluesky DID, Expo Push Token, notification settings (only when push notifications are enabled)
  • Expo Push Notification Service: Expo Push Token, notification title and body (when sending push notifications)
  • Bluesky Jetstream (Official): No data sent (receives events only)

5. Push Notification Tokens

When Users enable push notifications, the following information is sent to and stored on the Operator's server (Railway):

  • Expo Push Token (a notification delivery token issued by the Expo Push Notification service)
  • Bluesky DID (a public identifier used to identify the notification recipient)
  • Notification type settings (ON/OFF for likes, replies, mentions, reposts, and follows)

This information is used solely for push notification delivery. It is deleted from the server when push notifications are disabled or the App is uninstalled.

6. Feedback Information

When Users use the in-app feedback feature, the following information is sent:

  • Feedback type (bug report, feature request, dictionary correction, etc.)
  • Feedback content (subject, description, comments)
  • Related word information or post URLs (if applicable)

Article 3 (Purpose of Information Use)

The purposes for which the Operator uses information are as follows:

  1. To provide and operate this App
  2. For user authentication and linking with Bluesky accounts
  3. To provide word data storage and restoration functions
  4. To provide learning support functions such as dictionary search, translation, and text analysis
  5. To respond to user feedback and improve this App
  6. To fix bugs and improve features of this App
  7. To send push notifications (Bluesky social action notifications and learning reminders)

Article 4 (Provision of Personal Information to Third Parties)

  1. The Operator will not provide personal information to third parties without prior consent of the User, except in the following cases:
    • When necessary to protect the life, body, or property of a person and it is difficult to obtain consent
    • When particularly necessary to improve public health or promote the sound development of children and it is difficult to obtain consent
    • When it is necessary to cooperate with a national organ or local government in executing affairs prescribed by law and obtaining consent may impede such execution
    • Other cases permitted by law
  2. This App sends data to external services (DeepL, Yahoo! JAPAN, etc.) configured by Users, but this is based on API keys set by Users themselves and does not constitute third-party provision by the Operator.

Article 5 (Security Measures)

  1. The Operator takes the following measures for security management of personal information:
    • Authentication tokens and API keys are stored in the device's encrypted storage (Expo SecureStore)
    • Bluesky authentication uses OAuth (PKCE + DPoP) and does not obtain or store user passwords
    • Communication with external services uses HTTPS encrypted communication

Article 6 (Analytics and Tracking)

This App does not use analytics tools, advertising SDKs, or tracking tools. No features for collecting or analyzing user behavioral data are included.

Article 7 (Device Access Permissions)

This App may request access permissions to the following device features:

Permission Purpose
Photo Library To save and share learning progress images, and to attach images to posts
Camera To take photos for attaching to posts
Notifications To receive Bluesky social action notifications and learning reminders

These permissions are only requested when the User uses the corresponding features, and the basic features of this App can be used without granting them.

Article 8 (Data Deletion)

  1. Users can delete all learning data stored on the device by executing "Delete All Data" from the Settings screen of this App.
  2. Logging out will delete authentication information stored on the device.
  3. Deletion of data synced to PDS is left to the User's own PDS management.
  4. Uninstalling this App will delete all data on the device.
  5. Deleted data cannot be recovered. Users should use the data export function to create backups before deletion as needed.

Article 9 (Use by Minors)

This App is not specifically targeted at minors, but it is recommended that minors use it with parental consent.

Article 10 (Changes to Privacy Policy)

  1. The Operator may change this Policy as needed.
  2. Changes to this Policy will be notified within this App or on the Operator's website.
  3. If a User continues to use this App after changes, the User shall be deemed to have agreed to the revised Policy.
