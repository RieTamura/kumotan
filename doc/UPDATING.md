# ドキュメント更新チェックリスト

このドキュメントは、コードベースに変更を加えた際に更新が必要なドキュメントを追跡するためのチェックリストです。

## 🎯 目的

コードとドキュメントの同期を維持し、ドキュメントの陳腐化を防ぐことを目的としています。

## 🛠️ ドキュメント作成のベストプラクティス

### 1. 明確で簡潔に

- 技術用語は最小限に
- ステップバイステップの手順を提供
- 実例とコードスニペットを含める

### 2. 最新性の維持

- コード変更と同時にドキュメントを更新
- 古い情報は削除または非推奨としてマーク
- 日付とバージョンを記録

### 3. 一貫性

- 用語の統一（例：「単語帳」vs「ワードリスト」）
- フォーマットの統一（見出しレベル、コードブロック）
- ファイル命名規則の統一

### 4. アクセシビリティ

- 目次の提供（長いドキュメント）
- リンクを適切に設定
- 検索しやすいキーワードを使用

## 🔍 レビューチェックポイント

コード変更をコミットする前に、以下を確認してください：

1. **関連ドキュメントを特定**
   - この変更はどのドキュメントに影響しますか？
   - 新しいドキュメントが必要ですか？

2. **ドキュメントを更新**
   - 説明は正確で最新ですか？
   - 例やスクリーンショットは正しいですか？
   - リンクは切れていませんか？

3. **一貫性を確認**
   - 用語は統一されていますか？
   - 他のドキュメントと矛盾していませんか？

4. **他の開発者に確認**
   - ドキュメントを読んで理解できますか？
   - 手順に従って実行できますか？

## 📊 ドキュメント品質メトリクス

### 目標

- ドキュメントカバレッジ：主要機能100%
- ドキュメント最終更新：コード変更後1週間以内
- リンク切れ：0件
- スクリーンショット鮮度：3ヶ月以内

### 測定方法

- 毎週のレビューで確認
- プルリクエスト時のチェックリスト確認
- マイルストーン完了時の総合レビュー
